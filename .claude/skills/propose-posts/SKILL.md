# Propose Posts

- Read `CONTENT_STRATEGY.md` first.
- Ground the content in the actual recent work:
  - current conversation
  - recent repo changes
  - recent milestone, blocker, decision, or lesson
- Prefer one LinkedIn draft in French and one X draft in English unless the user asks for a different format.
- Use this structure:
  - angle
  - draft
  - suggested visual
  - optional X follow-up replies
- Keep the posts:
  - personal
  - technical
  - concrete
  - honest about tradeoffs
- understated
- calm
- low-ego
- Do not force a post when the recent work is too small or uninteresting.
- Prefer:
  - “I thought X, real testing showed Y”
  - “This looked simple, it wasn’t”
  - “I changed the architecture because...”
- French LinkedIn phrasing like:
  - “Je pensais que X. En pratique, Y.”
  - “Ça paraissait simple. Ça ne l’était pas.”
  - “J’ai changé d’avis sur ce point.”
- Avoid:
  - generic startup language
  - inflated claims
  - vague “big update” posts with no real lesson
  - corporate-builder tone
  - motivational fluff
  - trying to sound impressive

For Fall Guardian specifically, good hooks are:

- real-device validation
- simulator vs real-world behavior
- fall detection vs safety UX
- cross-platform parity
- backend reliability
- caregiver-app direction
- Android local SMS as fallback only
- production tradeoffs and architecture pivots

The target voice is:

- senior software engineer
- pragmatic
- boring-tech-positive
- “nothing to prove”
