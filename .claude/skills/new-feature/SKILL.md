# New Feature

Workflow:

1. Identify touched projects and cross-project contracts.
2. Read root and project-local `AGENTS.md`.
3. Preserve alert, invite, device, push, acknowledgement, cancellation, and
   history contracts.
4. Implement in the project that owns the behavior.
5. Add useful tests for behavior and edge cases.
6. Run relevant quality checks.
7. Document new commands, configuration, or user-visible flows when needed.
